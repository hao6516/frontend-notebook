# React有哪些常用的Hooks

# 1. useState

来声明状态变量

初始化

const [state, setState] = useState(initialState)

更新(两种更新方式)

  1. 传递更新函数（强烈推荐）
setAge(age => age + 1)

React会将更新函数放入队列中，然后在下一次渲染期间，它将按照相同的顺序调用他们

  1. 直接传递下一个状态
setAge(age + 1)

此时如果执行下面函数，初始age为25

const change = () => {
    setAge(age + 1)
    setAge(age + 1)
    setAge(age + 1)
}

会发现结果为26

# 2. useReducer

类似于redux，其实可以看成一个小型的redux

场景：如果状态变得越来约复杂，useState变得非常繁琐时，使用useReducer来管理状态，可以使得状态更新更加可控，可预测。

代码：

import { useReducer } from "react"
// 1.定义一个reducer函数（根据不同的action返回不同的新状态）
// 2.在组件中调用useReducer，并传入reducer函数和状态的初始值
// 3.事件发生时，通过dispatch函数分派一个action对象（通知reducer要返回哪个新状态并渲染UI）
function reducer(state: number, action: Action) {
  switch (action.type) {
    case "INCR":
      return state + 1
    case "DEC":
      return state - 1
    default:
      return state
  }
}

export default function Test() {
  const [state, dispatch] = useReducer(reducer, 0)
  return (
    <div>
      <h1>{state}</h1>
      <button onClick={() => dispatch({type: "INCR"})}>+</button>
      <button onClick={() => dispatch({type: "DEC"})}>-</button>
    </div>
  )
}

type Action = { type: "INCR" } | { type: "DEC" }

# 4. useMemo

官方解释：跳过代价昂贵的重新计算。即组件每次重新渲染的时候缓存计算结果

小栗子🌰：求0~n的和，n使用useState定义 先定义一个求和函数

const getSum = (count: number) => {
  let sum = 0
  for (let i = 0; i < count; i++) {
    sum += i
  }
  return sum
}

使用useMemo

export default function Test() {
  const [count, setCount] = useState(100)
  const [sum, setSum] = useState(0)
  const res = useMemo(() => {
    return getSum(count)
  },[count])
  return (
    <>
      <div>
        {res}
      </div>
      <button onClick={() => setCount(count+1)}>count+1</button>
      <div>
        {sum}
      </div>
      <button onClick={() => setSum(sum+1)}>sum+1</button>
    </>
  )
}

如果count不发生变化，useMemo所包裹的回调函数不会重复执行

不使用useMemo

export default function Test() {
  const [count, setCount] = useState(100)
  const [sum, setSum] = useState(0)
  const res = getSum(count)
  return (
    <>
      <div>
        {res}
      </div>
      <button onClick={() => setCount(count+1)}>count+1</button>
      <div>
        {sum}
      </div>
      <button onClick={() => setSum(sum+1)}>sum+1</button>
    </>
  )
}

此时getSum函数会被重复调用，造成极大的性能损耗

# 5. useContext

一般与createContext结合使用，用来跨级组件传递消息

官网🌰：

import { createContext, useContext } from 'react';

const ThemeContext = createContext(null);

export default function MyApp() {
  return (
    <ThemeContext.Provider value="dark">
      <Form />
    </ThemeContext.Provider>
  )
}

function Form() {
  return (
    <Panel title="Welcome">
      <Button>Sign up</Button>
      <Button>Log in</Button>
    </Panel>
  );
}

function Panel({ title, children }) {
  const theme = useContext(ThemeContext);
  const className = 'panel-' + theme;
  return (
    <section className={className}>
      <h1>{title}</h1>
      {children}
    </section>
  )
}

function Button({ children }) {
  const theme = useContext(ThemeContext);
  const className = 'button-' + theme;
  return (
    <button className={className}>
      {children}
    </button>
  );
}

不懂的地方：如果跨级组件不在一个jsx文件内的话，怎么实现传递消息？

答案：

在父组件中将creatContext这一行导出

//父组件
export const ThemeContext = createContext(null);

子组件内接收，然后就可以在子组件中使用了

//子组件
import { ThemeContext } from '父组件地址'

const Son = () => {
    const them = useContext(ThemeContext)
    return (
        ......
    )
}

# 6. useEffect

用于在React组件中创建不是由渲染本身引起的操作，比如发送AJAX请求，更改DOM等等

useEffect(() => { }, [])

参数1是一个函数，可以叫做副作用函数，在函数内部可以放置要执行的操作 参数2是一个数组，在数组里放置依赖项，不同依赖项会影响第一个参数函数的执行，当是一个空数组的时候，副作用函数只会在组件渲染完毕之后执行一次

const Child = forwardRef((props, ref) => {

const Father = () => {
  const childRef = useRef()

    childRef.current.sayHello()
  }

      <button onClick={() => click()}>click</button>
    </>
  )
}

# 9. useLayoutEffect

是useEffect的一个版本，在浏览器重新绘制屏幕之前触发，与useEffect的区别主要是useLayoutEffect会阻塞渲染，官网举出了一个toolTip的例子，通过使用useLayoutEffect，在dom更新后、页面渲染前，获取高度，然后重新渲染到合适的位置上去。

# 自定义Hook函数

概念:自定义Hook是以use打头的函数，通过自定义Hook函数可以实现逻辑的封装和复用

  1. 声明一个以use打头的函数
  2. 在函数内封装可复用的逻辑（只要是可复用的逻辑）
  3. 把组件中用到的状态或者回调return出去（以对象或者数组）
  4. 在每个组件中要用到这个逻辑，执行这个函数，解构出来状态或者回调进行使用

# Hook使用规则

  1. 只能在组件中或者其他自定义Hook函数中调用
  2. 只能在组件的顶层调用，不能嵌套在if、for、其他函数中
